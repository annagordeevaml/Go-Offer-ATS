# Настройка регистрации без подтверждения email

## Быстрая настройка

Чтобы регистрация работала без подтверждения email (просто логин и пароль):

### 1. Включите Email Provider

1. Откройте Supabase Dashboard: https://app.supabase.com
2. Выберите ваш проект
3. Перейдите в **Authentication** → **Providers**
4. Найдите **Email** provider
5. **Включите** его (нажмите "Enable")

### 2. Отключите подтверждение email

1. В Supabase Dashboard перейдите в **Authentication** → **Settings**
2. Найдите опцию **"Enable email confirmations"**
3. **Отключите** эту опцию (переключатель должен быть выключен)
4. Сохраните изменения

### 3. Настройте URL (если еще не сделано)

1. Перейдите в **Authentication** → **URL Configuration**
2. Установите **Site URL**: `http://localhost:3000`
3. Добавьте в **Redirect URLs**: `http://localhost:3000/**`
4. Сохраните

## Как это работает

После настройки:
- ✅ Пользователь регистрируется с email и паролем
- ✅ Сразу получает доступ (без подтверждения email)
- ✅ Может сразу войти в систему
- ✅ Никаких писем не отправляется

## Проверка

1. Попробуйте зарегистрироваться с новым email
2. После регистрации вы должны сразу попасть в приложение
3. Проверьте Supabase Dashboard → Authentication → Users - должен появиться новый пользователь

## Важно

- Email provider должен быть **включен** (иначе регистрация не будет работать)
- Email confirmations должны быть **отключены** (чтобы не требовать подтверждения)
- Это настройка для разработки/тестирования. Для production рекомендуется включить подтверждение email для безопасности.
